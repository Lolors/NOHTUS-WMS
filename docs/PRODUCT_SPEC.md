# Product Specification

## 목적

NOHTUS WMS는 물류팀이 실제 재고를 빠르게 찾고, 입고/출고/위치이동/재고실사/제품매칭을 처리하기 위한 웹 기반 WMS입니다.

## 주요 개념

### 표준제품명

WMS 내부에서 사용하는 기준 제품명입니다. 모든 재고, 출고, 이력은 표준제품명 기준으로 저장합니다.

### ERP명

사업장별 ERP에서 사용하는 제품명입니다. 매출/매입 등록에는 ERP명이 필요하므로 출고지시서에는 해당 사업장 ERP명을 비고에 표시합니다.

### 사업장

- 노투스팜
- 노투스
- NOH
- 비자료
- 등록대기(입고등록 전용)

## 입고 등록

### 기능

- 매입처 입력
- 사업장 선택
- 제품명/ERP명 검색
- 표준제품명 선택
- LOT/제조번호 입력
- 유통기한 입력
- 도면 클릭으로 위치 선택
- 입고 저장

### 저장 규칙

- 제품명은 표준제품명으로 저장
- 사업장은 표시 문자열이 아니라 실제 사업장명만 저장
  - 예: `노투스팜 (30 EA)` 표시 → `노투스팜` 저장
- 등록대기는 입고에서만 선택 가능

## 로케이션맵

### 기능

- 위치별 재고 확인
- 위치 상세 보기
- 제품 상세 보기
- 표준제품명 클릭 시 제품검색 자동 실행

### 표시 규칙

- 재고 0인 위치는 숨길 수 있음
- 제품 상세 이미지가 없으면 placeholder 표시
- 제품 상세 이미지는 150×150

## 제품검색

### 기능

- 표준제품명, ERP명, 비자료명, 별칭 기반 검색
- 제품 검색 결과 카드 표시
- 총재고 표시
- 제품 이미지 표시

### 이미지

- 제품검색 카드: 250×250
- 없으면 `photo_placeholder.svg`

## 출고지시

### 기능

- 제품 검색
- 장바구니 추가
- 요청수량 입력
- 저장
- 수정
- 취소
- 엑셀/PDF 출력

### 규칙

- 요청수량이 현재고보다 많으면 저장 금지
- 저장 시 현재고 차감
- 취소 시 재고 복원
- 수정 시 변경분만 재계산
- 출력 시 피킹동선 고려
- 비고에 사업장별 ERP명 표시

## 재고실사

### 기능

- 재고 실사용 엑셀 다운로드
- 기준재고 업로드
- 재고조정

### 조정 규칙

- 조정 후 이력에 최종재고 기록
- 최종재고는 같은 표준제품명 전체 총재고

## 이력 조회

### 최종재고 정의

해당 거래 직후의 같은 표준제품명 전체 총재고입니다. 실시간 재계산값이 아닙니다.

## 제품 매칭 관리

### 기능

- 제품매칭표 업로드
- 제품매칭표 수정
- 제품 매칭 정보 추가
- ERP명/비자료명/표준제품명 관리

## 제품 이미지

### 규칙

- 250×250 JPG 저장
- 원본 저장 안 함
- 표준제품명 기반 파일명
- 없으면 SVG placeholder
