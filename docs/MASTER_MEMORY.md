# MASTER MEMORY - NOHTUS WMS

이 문서는 ChatGPT와 개발자가 NOHTUS WMS의 맥락을 잃지 않도록 만든 기준 문서입니다.

## 회사/업무 맥락

NOHTUS WMS는 노투스팜 물류팀이 재고 위치, 입고, 출고, 사업장별 재고, 비자료 재고, 제품명 매칭을 관리하기 위해 만든 사내 웹앱입니다.

## 사업장

기본 사업장은 다음 4개입니다.

- 노투스팜
- 노투스
- NOH
- 비자료

입고 등록에서만 추가로 `등록대기`를 사용할 수 있습니다. 다른 메뉴에서는 `등록대기`로 이동하는 선택지를 만들지 않습니다. 단, 등록대기 재고가 존재하면 로케이션맵/검색에서는 출력될 수 있습니다.

## 제품명 원칙

DB의 재고 기준 제품명은 `표준제품명`입니다.

제품매칭표에는 다음 컬럼을 둡니다.

- 표준제품명
- 노투스팜 ERP명
- 노투스팜 ERP 제품코드
- NOH ERP명
- NOH ERP 제품코드
- 노투스 ERP명
- 비자료명
- 별칭

입고/기준재고/ERP명 검색 시 ERP명을 입력해도 최종 저장은 표준제품명으로 합니다.

## 로케이션 규칙

기본 포맷은 `A1-01-01`입니다.

- 첫 구간: 구역
- 둘째 구간: 라인
- 셋째 구간: 단

특수 위치:

- REC
- Q1 / Q2
- 홍보물랙
- 회색 카트
- 오른쪽 창고
- 사무실(4층)

## 입고 등록 핵심 기능

- 도면 클릭 시 입고 위치의 구역/라인/단이 즉시 연동됩니다.
- 이 기능은 절대 깨지면 안 됩니다.
- `__입고도면적용` 버튼/문구가 잠깐 보이는 문제가 오래된 버전부터 존재합니다.
- 이 문제는 `bug/inbound-map-placeholder` 브랜치에서 단독 해결합니다.

## 로케이션맵 핵심 기능

- 위치 상세 보기
- 상세보기에서 표준제품명 클릭
- 제품검색 자동 실행
- 검색 결과 출력

이 흐름은 코어 기능입니다.

## 출고 원칙

- 출고지시 저장 시 현재고를 차감합니다.
- 요청수량이 현재고보다 많으면 저장하지 않습니다.
- 출고지시서 출력 시 피킹동선을 고려합니다.
- 출고지시서 비고에는 해당 사업장 기준 ERP명을 표시합니다.

## 이력 조회 최종재고 원칙

`최종재고`는 실시간 현재 재고가 아니라, 해당 이력이 발생한 직후의 같은 표준제품명 전체 총재고입니다.

계산 기준:

- 사업장 무시
- LOT 무시
- 유통기한 무시
- 로케이션 무시
- 표준제품명만 기준

단, 조회 시점에 다시 계산하면 안 되고, 거래 발생 직후 `transactions.final_stock`에 저장되어야 합니다.

## 제품 이미지 원칙

- 기본 저장 크기: 250×250
- 제품검색 카드 표시: 250×250
- 로케이션맵 오른쪽 제품 상세 표시: 150×150로 축소 표시
- 기본 placeholder는 `assets/photo_placeholder.svg`
- 원본 이미지는 보관하지 않습니다.
- 파일명은 표준제품명 기반 안전화 이름을 사용합니다.

## 개발 철학

- 한 브랜치에서는 한 가지 목적만 수행합니다.
- 코어 도면/지도/검색 JS를 건드릴 때는 반드시 별도 bug/refactor 브랜치를 사용합니다.
- UI 수정과 기능 수정을 섞지 않습니다.
- 기능 추가보다 안정성이 우선입니다.
