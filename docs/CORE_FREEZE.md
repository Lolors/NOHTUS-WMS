# Core Freeze Policy

NOHTUS WMS에서 다음 기능은 안정성이 가장 중요합니다. 관련 코드는 리팩토링 전까지 블랙박스로 취급합니다.

## 절대 함부로 수정하지 않는 영역

### 1. 입고도면 연동

- 도면 클릭
- 입고 위치 구역/라인/단 반영
- 관련 session_state key
- 관련 JS 이벤트

### 2. 로케이션맵

- SVG/HTML 지도 렌더링
- 위치 상세보기
- 제품 상세보기 버튼
- 제품명 클릭 이벤트

### 3. 제품검색 자동 실행

- 로케이션맵 상세보기에서 제품명 클릭
- 제품검색 화면으로 전환
- 검색어 전달 방식

## 수정이 필요한 경우

다음 브랜치만 사용합니다.

- `bug/inbound-map-*`
- `bug/location-map-*`
- `refactor/core-*`

## 금지사항

- UI 브랜치에서 도면 JS 수정 금지
- 사진 기능 브랜치에서 로케이션맵 JS 수정 금지
- 출고 기능 브랜치에서 입고도면 세션키 수정 금지

## 현재 알려진 Core 관련 버그

- `__입고도면적용` 문구/버튼이 입고등록 진입 시 잠깐 노출됨
- 오래된 RC2.62 이후부터 존재한 것으로 확인
- 해결 브랜치: `bug/inbound-map-placeholder`
