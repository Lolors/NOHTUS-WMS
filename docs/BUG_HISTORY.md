# Bug History

## BUG-001: `__입고도면적용` 문구/버튼 노출

### 상태

조사 중

### 발견

RC2.62 이후부터 입고등록 메뉴 진입 시 타이틀 아래에 `__입고도면적용`이 잠깐 나타났다가 사라지는 현상 확인.

### 영향

- 사용자에게 디버그/내부 버튼처럼 보여 UX가 나쁨
- 일부 버전에서는 이를 숨기거나 제거하다가 도면 클릭 → 입고 위치 연동이 깨짐

### 가설

- Streamlit `st.button("__입고도면적용")` 또는 placeholder가 렌더링됨
- 이후 JS 또는 rerun으로 사라짐
- 버튼 자체가 도면 클릭 이벤트와 연결되어 있을 가능성 있음

### 해결 원칙

- 버튼 생성 로직을 무작정 삭제하지 않음
- 도면 클릭 연동이 유지되는지 확인
- 별도 브랜치 `bug/inbound-map-placeholder`에서 단독 수정

### 테스트

- 입고등록 진입 시 문구가 보이지 않아야 함
- 도면 클릭 시 위치 연동이 유지되어야 함

## BUG-002: 도면 수정 중 입고 위치 연동 깨짐

### 상태

과거 여러 RC에서 발생

### 원인

도면 렌더링/placeholder/button/JS를 함께 수정하면서 session_state 또는 widget key 연동이 깨진 것으로 추정.

### 교훈

도면 영역은 코어로 동결하고 전용 브랜치에서만 수정.
